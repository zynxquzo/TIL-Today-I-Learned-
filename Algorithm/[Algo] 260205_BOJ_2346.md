# 오늘의 알고리즘 (2026.02.05)

## 오늘 푼 문제

* **백준 2346번**: [풍선 터뜨리기](https://www.acmicpc.net/problem/2346) (Silver 3)

---

## 내 풀이
```python
from collections import deque

N = int(input())  # 풍선의 개수
balloon_paper = list(map(int, input().split()))  # 각 풍선 안의 종이에 적힌 숫자
q = deque()

# 큐에 (풍선번호, 종이숫자) 튜플 형태로 저장
for idx, paper in enumerate(balloon_paper):
    q.append((idx+1, paper))  # 풍선 번호는 1부터 시작

while q:
    balloon_num, paper = q.popleft()  # 맨 앞 풍선 터뜨리기
    print(balloon_num, end=' ')  # 터뜨린 풍선 번호 출력
    
    if not q:  # 마지막 풍선이면 종료
        break
    
    # paper 값에 따라 이동
    if paper > 0:  # 양수일 때: 오른쪽으로 이동
        for i in range(paper - 1):  # paper-1번 이동
            q.append(q.popleft())
    else:  # 음수일 때: 왼쪽으로 이동
        for i in range(abs(paper)):  # abs(paper)번 이동
            q.appendleft(q.pop())
```

## 새로 배운 개념

### 1. 튜플 언패킹(Unpacking)

* `balloon_num, paper = q.popleft()`처럼 튜플을 여러 변수로 한 번에 나눠 받을 수 있다.
* `(1, 3)` 튜플에서 `balloon_num = 1`, `paper = 3`으로 자동 분리된다.
* `enumerate()`에서도 `for idx, paper in enumerate(list):`처럼 언패킹을 사용한다.
* 코드가 훨씬 간결해지고 가독성이 좋아진다.

### 2. deque의 양방향 연산

* `popleft()`: 맨 앞 요소 제거
* `pop()`: 맨 뒤 요소 제거
* `append()`: 맨 뒤에 추가
* `appendleft()`: 맨 앞에 추가
* 양방향으로 자유롭게 요소를 추가/제거할 수 있어서 원형 큐 구현에 적합하다.

### 3. 원형 큐에서의 이동 개념

* **양수 이동 (오른쪽)**: 현재 위치를 포함해서 세기 때문에 `paper - 1`번 이동한다.
  - 예: 3칸 이동 = 1칸(현재), 2칸, 3칸 → 실제로는 2번 회전
* **음수 이동 (왼쪽)**: 뒤에서부터 세기 때문에 `abs(paper)`번 그대로 이동한다.
  - 예: -3칸 = 뒤에서 1번째, 2번째, 3번째 → 3번 회전
* 이 차이를 이해하는 것이 문제의 핵심이었다.

### 4. abs() 함수 활용

* `abs()`는 절댓값을 반환하는 함수다.
* `abs(-3) = 3`, `abs(5) = 5`
* 음수를 양수로 바꿔서 반복 횟수로 사용할 때 유용하다.
* `-paper`로도 가능하지만 `abs()`가 의미가 더 명확하다.

### 5. 시뮬레이션 디버깅 방법

* 손으로 직접 예제를 따라가며 각 단계를 확인하는 것이 중요하다.
* "왜 -1을 하는가?"처럼 작은 부분까지 정확히 이해해야 한다.
* 출력 형식(`end=' '`)도 놓치기 쉬운 부분이다.

## 회고

* 처음에는 1966번 문제와 비슷하다고 생각해서 쉽게 접근했지만, 원형 큐의 이동 개념이 생각보다 복잡했다.
* 양수와 음수 이동에서 왜 -1을 하는지/안 하는지 이해하는 데 시간이 걸렸지만, 그림으로 그려보니 명확하게 이해할 수 있었다.
* 튜플 언패킹 문법이 처음에는 낯설었지만, 한 번 이해하니 코드가 훨씬 깔끔해진다는 것을 느꼈다.
* `popleft()`, `pop()` 함수를 호출할 때 괄호 `()`를 빼먹는 실수를 했는데, 함수는 반드시 괄호와 함께 호출해야 한다는 것을 다시 배웠다.
* 비슷한 유형의 문제(1966번)를 먼저 풀어본 경험이 큰 도움이 되었다. deque와 튜플을 활용하는 패턴을 익힐 수 있었다.

# 오늘의 알고리즘 (2026.03.07)

## 오늘 푼 문제

* **백준**: [2606번 바이러스](https://www.acmicpc.net/problem/2606) (Silver 3)

---

## 내 풀이
```python
import sys

n = int(sys.stdin.readline())  # 컴퓨터 개수
m = int(sys.stdin.readline())  # 연결 쌍의 개수

graph = {}

# 그래프 초기화
for i in range(1, n+1):
    graph[i] = []
    
# 연결 정보 입력
for _ in range(m):
    a, b = map(int, sys.stdin.readline().split())
    graph[a].append(b)
    graph[b].append(a)

visited = []

# DFS 함수
def dfs(node):
    visited.append(node)
    for next_node in graph[node]:
        if next_node not in visited:
            dfs(next_node)

# 1번부터 시작
dfs(1)

# 1번 제외하고 출력
print(len(visited)-1)
```

## 새로 배운 개념

### 1. 그래프(Graph)란?

* **그래프**: 노드(점)와 간선(선)으로 연결된 구조
* **노드(Node)**: 정점, 이 문제에서는 컴퓨터
* **간선(Edge)**: 노드를 연결하는 선, 이 문제에서는 네트워크 연결
```python
# 그래프 시각화
1 --- 2 --- 3
|
5 --- 6

노드: 1, 2, 3, 5, 6
간선: 1-2, 2-3, 1-5, 5-6
```

### 2. 그래프 표현: 인접 리스트

* 딕셔너리를 사용해 각 노드의 연결 정보를 저장
* 키: 노드 번호, 값: 연결된 노드들의 리스트
```python
graph = {
    1: [2, 5],    # 1번은 2번, 5번과 연결
    2: [1, 3],    # 2번은 1번, 3번과 연결
    3: [2],       # 3번은 2번과 연결
    5: [1, 6],
    6: [5]
}
```

### 3. 양방향 그래프

* 이 문제는 **무방향 그래프** (양방향 연결)
* `1-2` 연결은 `1→2`와 `2→1` 모두 가능
* 따라서 양쪽 모두에 추가해야 함
```python
# 입력: 1 2 (1과 2가 연결됨)
graph[1].append(2)  # 1에서 2로 갈 수 있음
graph[2].append(1)  # 2에서 1로도 갈 수 있음
```

### 4. DFS (깊이 우선 탐색)

* **Depth First Search**: 한 방향으로 깊게 파고드는 탐색 방법
* 막다른 곳까지 가본 후, 돌아와서 다른 경로 탐색
* **재귀 함수** 또는 **스택**으로 구현
```python
# DFS 동작 방식
1 → 2 → 3 (끝) → 돌아와서 → 5 → 6 (끝)

# 방문 순서: 1, 2, 3, 5, 6
```

### 5. 재귀 함수 (Recursion)

* **재귀**: 함수가 자기 자신을 호출하는 것
* DFS를 재귀로 구현하면 코드가 간결해짐
```python
def dfs(node):
    visited.append(node)
    for next_node in graph[node]:
        if next_node not in visited:
            dfs(next_node)  # 자기 자신을 다시 호출!

# dfs(1) → dfs(2) → dfs(3) → ... (재귀 호출)
```

### 6. 방문 체크의 중요성

* 이미 방문한 노드를 다시 방문하면 **무한 루프** 발생
* `visited` 리스트로 방문한 노드를 기록
```python
# 방문 체크 없으면:
1 → 2 → 1 → 2 → 1 → ... (무한 반복!)

# 방문 체크 있으면:
1 → 2 (1은 이미 visited) → 3 (2는 이미 visited)
```

### 7. map 함수

* 여러 값에 같은 함수를 한 번에 적용
* 입력 처리할 때 자주 사용
```python
# "1 2"를 입력받아서 정수로 변환
a, b = map(int, input().split())
# input().split() → ['1', '2']
# map(int, ...) → int('1'), int('2')
# a, b = 1, 2
```

### 8. 문제 해결 흐름

**1단계: 그래프 초기화**
```
n개의 노드에 대해 빈 리스트 생성
graph[1] = []
graph[2] = []
...
```

**2단계: 연결 정보 입력**
```
m개의 연결 쌍 입력받기
각 쌍에 대해 양방향으로 추가
graph[a].append(b)
graph[b].append(a)
```

**3단계: DFS 탐색**
```
1번 노드부터 시작
연결된 모든 노드 방문
visited에 방문 기록
```

**4단계: 결과 계산**
```
visited 길이 - 1 (1번 제외)
```

### 9. DFS 재귀 호출 과정 상세 설명

**그래프 예시:**
```
1 --- 2 --- 3
|
5 --- 6
```

**실행 과정:**

```
[호출 1] dfs(1) 시작
  visited = [1]
  graph[1] = [2, 5] 확인
  2가 visited에 없음 → dfs(2) 호출
  
  [호출 2] dfs(2) 시작 (dfs(1)은 대기 중)
    visited = [1, 2]
    graph[2] = [1, 3] 확인
    1은 이미 visited → 건너뜀
    3이 visited에 없음 → dfs(3) 호출
    
    [호출 3] dfs(3) 시작 (dfs(2)도 대기 중)
      visited = [1, 2, 3]
      graph[3] = [2] 확인
      2는 이미 visited → 건너뜀
      더 이상 방문할 곳 없음
    [호출 3] dfs(3) 종료
    
  [호출 2] dfs(2) 복귀 및 종료
  
  [호출 1] dfs(1) 복귀
  5가 visited에 없음 → dfs(5) 호출
  
  [호출 4] dfs(5) 시작
    visited = [1, 2, 3, 5]
    graph[5] = [1, 6] 확인
    1은 이미 visited → 건너뜀
    6이 visited에 없음 → dfs(6) 호출
    
    [호출 5] dfs(6) 시작
      visited = [1, 2, 3, 5, 6]
      graph[6] = [5] 확인
      5는 이미 visited → 건너뜀
    [호출 5] dfs(6) 종료
    
  [호출 4] dfs(5) 종료
  
[호출 1] dfs(1) 종료

최종 visited = [1, 2, 3, 5, 6]
```

### 10. 예시로 이해하기

**입력:**
```
7        컴퓨터 개수
6        연결 쌍의 개수
1 2
2 3
1 5
5 2
5 6
4 7
```

**그래프 구조:**
```
graph = {
    1: [2, 5],
    2: [1, 3, 5],
    3: [2],
    4: [7],
    5: [1, 2, 6],
    6: [5],
    7: [4]
}
```

**시각화:**
```
1 --- 2 --- 3
|    /
|   /
5 ---
|
6

4 --- 7 (분리됨)
```

**DFS 실행:**
```
dfs(1) 시작
├─ visited = [1]
├─ 2 방문 → dfs(2)
│  ├─ visited = [1, 2]
│  ├─ 1은 이미 방문
│  ├─ 3 방문 → dfs(3)
│  │  ├─ visited = [1, 2, 3]
│  │  └─ dfs(3) 종료
│  ├─ 5 방문 → dfs(5)
│  │  ├─ visited = [1, 2, 3, 5]
│  │  ├─ 1, 2는 이미 방문
│  │  ├─ 6 방문 → dfs(6)
│  │  │  ├─ visited = [1, 2, 3, 5, 6]
│  │  │  └─ dfs(6) 종료
│  │  └─ dfs(5) 종료
│  └─ dfs(2) 종료
└─ 5는 이미 방문
└─ dfs(1) 종료

visited = [1, 2, 3, 5, 6]
감염된 컴퓨터 = 5 - 1 = 4대
```

**최종 출력:** `4`

### 11. 주요 실수 포인트

* **그래프 초기화 누락**: 모든 노드에 대해 빈 리스트를 만들지 않으면 KeyError 발생
```python
# 잘못된 코드
graph = {}
graph[1].append(2)  # KeyError! graph[1]이 없음

# 올바른 코드
graph = {}
for i in range(1, n+1):
    graph[i] = []  # 미리 초기화
```

* **양방향 추가 누락**: 한 쪽만 추가하면 한 방향으로만 탐색 가능
```python
# 잘못된 코드
graph[a].append(b)  # a → b만 가능

# 올바른 코드
graph[a].append(b)  # a → b
graph[b].append(a)  # b → a (양방향)
```

* **방문 체크 누락**: visited 확인 없이 재귀하면 무한 루프
```python
# 잘못된 코드
def dfs(node):
    visited.append(node)
    for next_node in graph[node]:
        dfs(next_node)  # 무한 재귀!

# 올바른 코드
def dfs(node):
    visited.append(node)
    for next_node in graph[node]:
        if next_node not in visited:  # 방문 체크!
            dfs(next_node)
```

* **1번 포함해서 출력**: 문제는 1번을 제외한 개수를 요구함
```python
# 잘못된 출력
print(len(visited))  # 5 (1번 포함)

# 올바른 출력
print(len(visited) - 1)  # 4 (1번 제외)
```

### 12. DFS vs BFS 비교

**DFS (깊이 우선 탐색):**
- 한 방향으로 쭉 파고듦
- 재귀 또는 스택 사용
- 메모리 효율적 (경로 하나만 기억)
- 구현이 간단 (재귀 사용 시)

**BFS (너비 우선 탐색):**
- 가까운 것부터 차례대로
- 큐 사용
- 최단 거리 보장
- 메모리 사용량 많음 (같은 레벨 모두 기억)

**이 문제에서는:**
- 최단 거리가 필요 없음
- 단순히 "연결된 모든 노드" 찾기
- **DFS, BFS 둘 다 가능!**

## 회고

* **방문 체크의 중요성**: 무한 루프를 방지하기 위해 반드시 필요하다.
* **양방향 그래프**: 양쪽 모두에 추가해야 한다는 것을 배웠다.

# 오늘의 알고리즘 (2026.02.02)

## 오늘 푼 문제

* **백준 1966번**: [프린터 큐](https://www.acmicpc.net/problem/1966) (Silver 3)

---

## 내 풀이
```python
from collections import deque  # 큐 자료구조 사용을 위한 import

T = int(input())  # 테스트 케이스 개수

for _ in range(T):  # 각 테스트 케이스마다 반복
    N, M = map(int, input().split())  # N: 문서 개수, M: 궁금한 문서의 위치
    importance = list(map(int, input().split()))  # 각 문서의 중요도 리스트
    q = deque()  # 프린터 큐 생성
    
    for idx, imp in enumerate(importance):  # 인덱스와 중요도를 함께 순회
        q.append((idx, imp))  # (원래 인덱스, 중요도) 튜플 형태로 큐에 저장
	
    count = 0  # 인쇄된 문서 수를 세는 변수
    
    while q:  # 큐가 빌 때까지 반복
        current = q[0]  # 큐의 맨 앞 문서 확인 (아직 제거하지 않음)
        current_importance = current[1]  # 현재 문서의 중요도 (튜플의 두 번째 값)
        max_importance = max(q, key=lambda x: x[1])[1]  # 큐에서 가장 높은 중요도 찾기
        
        if current_importance < max_importance:  # 뒤에 더 중요한 문서가 있는 경우
            q.popleft()  # 맨 앞 문서 제거
            q.append(current)  # 맨 뒤로 다시 추가
        else:  # 현재 문서가 가장 중요한 경우
            q.popleft()  # 문서 인쇄 (큐에서 제거)
            count += 1  # 인쇄 횟수 증가
            if current[0] == M:  # 인쇄된 문서가 우리가 찾던 M번째 문서인지 확인
                print(count)  # 몇 번째로 인쇄되었는지 출력
                break  # 답을 찾았으므로 while 문 종료
```

## 새로 배운 개념

### 1. 큐(Queue) 자료구조의 이해

* 큐는 "줄을 서는 것"처럼 먼저 들어온 것이 먼저 나가는 FIFO(First In First Out) 구조다.
* `popleft()`로 맨 앞 요소를 제거하고, `append()`로 맨 뒤에 요소를 추가한다.
* 프린터 대기열처럼 순서가 중요한 문제에 큐를 활용할 수 있다는 것을 배웠다.

### 2. enumerate() 함수 활용

* `enumerate()`는 리스트를 순회하면서 인덱스와 값을 동시에 가져올 수 있다.
* `for idx, imp in enumerate(importance):`처럼 사용하면 각 요소의 위치를 추적할 수 있다.
* M번째 문서를 추적하기 위해 원래 인덱스를 저장하는 데 유용했다.

### 3. lambda 함수와 key 파라미터

* `lambda x: x[1]`은 "x의 1번 인덱스 값을 반환하는 간단한 함수"를 한 줄로 표현한 것이다.
* `max(q, key=lambda x: x[1])`은 튜플 리스트에서 두 번째 값(중요도)을 기준으로 최댓값을 찾는다.
* lambda를 사용하면 복잡한 자료구조에서도 원하는 기준으로 비교할 수 있다.

### 4. 튜플을 활용한 데이터 추적

* `(인덱스, 중요도)` 형태의 튜플로 저장하면 두 가지 정보를 함께 관리할 수 있다.
* `current[0]`은 원래 위치, `current[1]`은 중요도를 의미한다.
* 큐에서 요소가 이동해도 원래 위치 정보를 잃지 않고 추적할 수 있었다.

### 5. 시뮬레이션 문제 해결 전략

* 문제에서 주어진 규칙을 그대로 코드로 구현하는 "시뮬레이션" 방식을 배웠다.
* 복잡해 보이지만 단계별로 조건을 나눠서 처리하면 해결할 수 있다.

## 회고

* 처음에는 큐가 무엇인지 정확히 이해하지 못했지만, "줄 서기"라는 비유로 개념을 이해할 수 있었다.
* M번째 문서를 어떻게 추적할지 고민했는데, 튜플로 원래 위치를 함께 저장하는 아이디어가 핵심이었다.
* `enumerate()`, `lambda`, `max()` 같은 파이썬 기능들을 조합해서 문제를 간결하게 해결할 수 있다는 것을 배웠다.
* 들여쓰기 실수로 코드가 제대로 동작하지 않았던 경험을 통해 파이썬에서 들여쓰기의 중요성을 다시 한번 깨달았다.
* 단순히 코드를 작성하는 것이 아니라 "왜 이렇게 동작하는가?"를 이해하는 것이 중요하다는 것을 느꼈다.

## 오늘 푼 문제

* **백준**: [11725번 트리의 부모 찾기](https://www.acmicpc.net/problem/11725) (Silver 2)

## 내 풀이
```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**6)

n = int(input())

graph = [[] for _ in range(n+1)]
visited = [False] * (n+1)
parent = [0] * (n+1)

for _ in range(n-1):
    a, b = map(int, input().split())
    graph[a].append(b)
    graph[b].append(a)

def dfs(now):
    visited[now] = True
    
    for next_node in graph[now]:
        if not visited[next_node]:
            parent[next_node] = now
            dfs(next_node)

dfs(1)

for i in range(2, n+1):
    print(parent[i])
```

## 새로 배운 개념

### 1. 트리(Tree) 자료구조란?

* **트리**: 계층적 구조를 가진 자료구조
* **핵심 특징**: 
  - 사이클이 없음 (순환하지 않음)
  - N개의 노드가 있으면 항상 N-1개의 간선
  - 모든 노드가 연결되어 있음
```
        1 (루트)
       /|\ 
      / | \
     6  4  2
    /   |
   3    7
  /
 5
```

### 2. 트리의 주요 용어

**루트(Root):**
- 가장 위에 있는 노드
- 이 문제에서는 항상 1번 노드

**부모(Parent) - 자식(Child):**
- 위에 있는 노드가 부모, 아래가 자식
- 6의 부모는 1, 3의 부모는 6

**간선(Edge):**
- 노드를 연결하는 선
- N개 노드 → N-1개 간선 (트리의 특성!)

### 3. 인접 리스트(Adjacency List)

* **인접 리스트**: 그래프의 연결 정보를 저장하는 방법
* **장점**: 메모리 효율적, 연결된 노드만 저장
```python
# n=7일 때
graph = [[] for _ in range(n+1)]

# 입력 1 6 처리 후
graph[1] = [6]
graph[6] = [1]

# 최종 결과 (예시)
graph = [
    [],           # 0번 (안 씀)
    [6, 4],       # 1번은 6, 4와 연결
    [4],          # 2번은 4와 연결
    [6, 5],       # 3번은 6, 5와 연결
    [1, 2, 7],    # 4번은 1, 2, 7과 연결
    [3],          # 5번은 3과 연결
    [1, 3],       # 6번은 1, 3과 연결
    [4]           # 7번은 4와 연결
]
```

### 4. 양방향 그래프 구성

* **양방향**: 두 노드가 서로 갈 수 있음
* **이유**: 입력에서 누가 부모인지 알 수 없기 때문
```python
# 입력: 1 6
graph[1].append(6)  # 1 → 6
graph[6].append(1)  # 6 → 1

# 양방향으로 저장해야 DFS로 탐색 가능!
```

### 5. DFS로 트리 탐색하기

**DFS(Depth First Search)의 핵심:**
- 한 방향으로 깊이 들어가며 탐색
- 재귀 함수로 구현
- 방문 체크로 무한 루프 방지
```python
def dfs(now):
    visited[now] = True  # 방문 체크
    
    for next_node in graph[now]:  # 연결된 노드 확인
        if not visited[next_node]:  # 미방문 노드만
            parent[next_node] = now  # 부모 기록!
            dfs(next_node)  # 재귀 호출
```

### 6. 부모 찾기의 핵심 아이디어

**1번(루트)부터 시작해서 연결된 노드 방문:**
- "내가 너를 방문했으니, 나는 너의 부모야!"
- 방문한 노드는 다시 방문 안 함
```python
# 1번 방문 → 6, 4 발견
parent[6] = 1  # 6의 부모는 1
parent[4] = 1  # 4의 부모는 1

# 6번 방문 → 3 발견 (1은 이미 방문함)
parent[3] = 6  # 3의 부모는 6

# 이런 식으로 모든 노드의 부모 기록!
```

### 7. DFS 실행 과정 상세

**입력 예시:**
```
7
1 6
6 3
3 5
4 1
2 4
4 7
```

**탐색 과정:**
```
dfs(1) 시작
├─ visited[1] = True
├─ graph[1] = [6, 4] 확인
│  
├─ next_node = 6 (미방문)
│  ├─ parent[6] = 1 기록
│  └─ dfs(6) 호출
│     ├─ visited[6] = True
│     ├─ graph[6] = [1, 3]
│     ├─ 1은 이미 방문 → 건너뛰기
│     ├─ next_node = 3 (미방문)
│     │  ├─ parent[3] = 6 기록
│     │  └─ dfs(3) 호출
│     │     ├─ visited[3] = True
│     │     ├─ graph[3] = [6, 5]
│     │     ├─ 6은 이미 방문 → 건너뛰기
│     │     ├─ next_node = 5 (미방문)
│     │     │  ├─ parent[5] = 3 기록
│     │     │  └─ dfs(5) 호출
│     │     │     └─ 더 이상 미방문 노드 없음
│     │     └─ dfs(3) 종료
│     └─ dfs(6) 종료
│  
└─ next_node = 4 (미방문)
   ├─ parent[4] = 1 기록
   └─ dfs(4) 호출
      ├─ visited[4] = True
      ├─ graph[4] = [1, 2, 7]
      ├─ 1은 이미 방문 → 건너뛰기
      ├─ next_node = 2 (미방문)
      │  ├─ parent[2] = 4 기록
      │  └─ dfs(2) 종료
      ├─ next_node = 7 (미방문)
      │  ├─ parent[7] = 4 기록
      │  └─ dfs(7) 종료
      └─ dfs(4) 종료

최종 parent = [0, 0, 4, 6, 1, 3, 1, 4]
                0  1  2  3  4  5  6  7
```

### 8. 재귀 깊이 제한 설정

* **Python 기본 재귀 깊이**: 1000
* **문제 상황**: 노드가 최대 100,000개 → 재귀 깊이 부족!
* **해결 방법**: `sys.setrecursionlimit(10**6)`
```python
import sys
sys.setrecursionlimit(10**6)  # 재귀 깊이를 100만으로 설정

# 이게 없으면 런타임 에러 발생!
```

### 12. DFS vs BFS 비교

**DFS (이 문제에서 사용):**
- 재귀 함수로 구현
- 깊이 우선으로 탐색
- 코드가 간결함
```python
def dfs(now):
    visited[now] = True
    for next_node in graph[now]:
        if not visited[next_node]:
            parent[next_node] = now
            dfs(next_node)
```

**BFS (이 문제에서도 가능):**
- 큐(queue)로 구현
- 너비 우선으로 탐색
- 최단 경로 찾을 때 유리
```python
from collections import deque

def bfs():
    queue = deque([1])
    visited[1] = True
    
    while queue:
        now = queue.popleft()
        for next_node in graph[now]:
            if not visited[next_node]:
                visited[next_node] = True
                parent[next_node] = now
                queue.append(next_node)
```

### 13. 주요 실수 포인트

* **재귀 깊이 제한 안 설정**: 런타임 에러!
```python
# 잘못된 코드
import sys
input = sys.stdin.readline
# sys.setrecursionlimit(10**6) 없음!

# 올바른 코드
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**6)  # 필수!
```

* **양방향 연결 안 함**: 일부 노드를 못 찾음
```python
# 잘못된 코드
graph[a].append(b)  # 단방향만

# 올바른 코드
graph[a].append(b)  # 양방향
graph[b].append(a)
```

* **방문 체크 안 함**: 무한 루프 발생
```python
# 잘못된 코드
def dfs(now):
    for next_node in graph[now]:
        parent[next_node] = now
        dfs(next_node)  # 무한 재귀!

# 올바른 코드
def dfs(now):
    visited[now] = True  # 방문 체크!
    for next_node in graph[now]:
        if not visited[next_node]:  # 미방문만
            parent[next_node] = now
            dfs(next_node)
```

* **간선 개수 실수**: n개가 아닌 n-1개!
```python
# 잘못된 코드
for _ in range(n):  # n번 반복

# 올바른 코드
for _ in range(n-1):  # n-1번 반복 (트리 특성)
```

### 14. 트리 문제 특징

**트리를 사용하는 문제의 신호:**
- "부모-자식 관계"
- "루트가 주어짐"
- "사이클 없음"
- "N개 노드, N-1개 간선"
- "계층 구조"

**트리 문제 풀이 전략:**
1. 인접 리스트로 그래프 구성
2. DFS/BFS로 탐색
3. 재귀 깊이 제한 설정 (Python)
4. 방문 체크로 무한 루프 방지

## 회고

* **트리의 핵심**: N개 노드는 항상 N-1개 간선을 가지며, 사이클이 없다.
* **부모 찾기 전략**: 루트부터 DFS로 탐색하며 방문하는 노드의 부모를 기록한다.
* **재귀 깊이 제한**: Python에서 깊은 재귀를 사용할 땐 `sys.setrecursionlimit()` 사용
* **양방향 저장**: 입력에서 부모-자식 관계를 모르므로 양방향으로 저장해야 한다.

## 오늘 푼 문제

* **백준**: [2747번 피보나치 수](https://www.acmicpc.net/problem/2747) (Bronze 2)

---

## 내 풀이
```python
# 백준 2747번 - 피보나치 수

import sys

n = int(sys.stdin.readline())

if n == 0:
    print(0)
elif n == 1:
    print(1)
else:
    a = 0
    b = 1
    for i in range(n-1):
        new = a + b
        a = b
        b = new
    print(b)
```

## 새로 배운 개념

### 1. 피보나치 수열의 정의

**피보나치 수열 규칙:**
- F(0) = 0
- F(1) = 1
- F(n) = F(n-1) + F(n-2) (n ≥ 2)

**수열 예시:**
```
F(0) = 0
F(1) = 1
F(2) = F(1) + F(0) = 1 + 0 = 1
F(3) = F(2) + F(1) = 1 + 1 = 2
F(4) = F(3) + F(2) = 2 + 1 = 3
F(5) = F(4) + F(3) = 3 + 2 = 5
```

### 2. 두 가지 접근 방식

**방법 1: 리스트에 모든 값 저장**
```python
fib = [0, 1]
for i in range(2, n+1):
    fib.append(fib[i-1] + fib[i-2])
print(fib[n])
```
- 장점: 모든 피보나치 수를 기록
- 단점: 메모리 사용량 많음 (O(n))

**방법 2: 변수 두 개만 사용 (채택한 방법)**
```python
a = 0  # F(n-2)
b = 1  # F(n-1)
for i in range(n-1):
    new = a + b  # F(n)
    a = b
    b = new
print(b)
```
- 장점: 메모리 효율적 (O(1))
- 단점: 최종 값만 알 수 있음
- **이 문제에서는 최종 값만 필요하므로 방법 2가 적합!**

### 3. 경우 분류의 필요성

**왜 n=0, n=1을 따로 처리할까?**
```python
if n == 0:
    print(0)
elif n == 1:
    print(1)
else:
    # n >= 2일 때만 for문 실행
```

**이유:**
- F(0) = 0, F(1) = 1은 이미 정의된 값
- for문은 "이전 두 값"을 더해서 새 값을 만드는 과정
- n=0, n=1일 때는 계산 없이 바로 출력 가능

**만약 분류하지 않으면?**
```python
# n=1일 때
a = 0, b = 1
for i in range(0):  # range(1-1) = range(0) → 반복 안 함
    # 실행 안 됨
print(b)  # 1 출력 → 우연히 맞음

# n=0일 때
a = 0, b = 1
for i in range(-1):  # range(0-1) = range(-1) → 반복 안 함
    # 실행 안 됨
print(b)  # 1 출력 → 틀림! (정답은 0)
```

→ n=0일 때 오답이 나오므로 경우 분류 필요!

### 4. 반복 횟수 계산

**n번째 피보나치 수를 구하려면 몇 번 반복?**
```
초기 상태: a=0(F0), b=1(F1)

n=2: F(2) 1번 계산 → 1번 반복
n=3: F(2), F(3) 2번 계산 → 2번 반복
n=4: F(2), F(3), F(4) 3번 계산 → 3번 반복
```

**패턴:** n ≥ 2일 때, **(n-1)번** 반복
```python
for i in range(n-1):  # n-1번 반복
```

### 5. 변수 업데이트 과정

**핵심: 슬라이딩 윈도우 방식**

한 단계씩 따라가기 (n=4 예시):
```
초기: a=0(F0), b=1(F1)

[1번째 반복]
new = a + b = 0 + 1 = 1 (F2)
a = b = 1 (F1)
b = new = 1 (F2)
→ 결과: a=1, b=1

[2번째 반복]
new = a + b = 1 + 1 = 2 (F3)
a = b = 1 (F2)
b = new = 2 (F3)
→ 결과: a=1, b=2

[3번째 반복]
new = a + b = 1 + 2 = 3 (F4)
a = b = 2 (F3)
b = new = 3 (F4)
→ 결과: a=2, b=3

최종: b=3 출력 → F(4)=3 ✓
```

### 6. 업데이트 순서의 중요성

**올바른 순서:**
```python
new = a + b  # 1. 먼저 새 값 계산
a = b        # 2. a를 한 칸 앞으로
b = new      # 3. b를 새 값으로
```

**잘못된 순서:**
```python
new = a + b
b = new      # b를 먼저 바꿈
a = b        # a도 new가 되어버림! (원래 b 값 소실)
```

**왜 순서가 중요한가?**
- `a = b`는 "원래 b의 값"을 a에 저장하는 것
- 만약 `b = new`를 먼저 하면, 원래 b 값이 사라짐
- 그 후 `a = b`를 하면 a도 new가 되어 다음 계산이 틀려짐

### 7. 변수 이름 선택

**가능한 변수명들:**
```python
# 간단한 방식
a, b, new

# 의미있는 방식
prev, curr, next
first, second, temp
```

**이 문제에서 선택한 방식:**
- `a`, `b`: 간단하고 짧아서 코드가 깔끔
- `new`: 새로 계산된 값임을 명확히 표현

### 8. 최종 출력값

**for문 종료 후:**
```python
print(b)  # b에 F(n)이 저장되어 있음
```

**왜 b를 출력?**
- 매 반복마다 `b = new`로 최신 피보나치 수를 b에 저장
- for문이 끝나면 b에 F(n)이 들어있음

**new를 출력하면?**
```python
print(new)  # 이것도 작동은 함
```
- 파이썬에서는 for문 안 변수가 밖에서도 유효
- 하지만 `b`가 더 명확한 의미 전달

### 9. 시간복잡도와 공간복잡도

**시간복잡도: O(n)**
- for문이 (n-1)번 반복
- 각 반복마다 덧셈 한 번 → O(1)
- 전체: O(n)

**공간복잡도: O(1)**
- 변수 a, b, new 세 개만 사용
- n이 커져도 메모리 사용량 일정

**리스트 방식과 비교:**
- 리스트: 공간복잡도 O(n)
- 변수: 공간복잡도 O(1)
- → 변수 방식이 메모리 효율적!

### 10. 주요 실수 포인트

**실수 1: 반복 횟수 착각**
```python
# 잘못된 코드
for i in range(n):  # n번 반복 → F(n+1)까지 계산

# 올바른 코드
for i in range(n-1):  # (n-1)번 반복 → F(n)까지 계산
```

**실수 2: 경우 분류 누락**
```python
# n=0일 때 틀린 답 출력
# n=0, n=1 경우를 반드시 따로 처리!
```

**실수 3: 변수 업데이트 순서**
```python
# 잘못된 순서
b = new
a = b  # 원래 b 값 소실!

# 올바른 순서
a = b
b = new
```

**실수 4: 리스트 문법 혼동**
```python
# 잘못된 코드
F = []
F(0) = 0  # 에러! 함수 호출 문법

# 올바른 코드 (리스트 방식)
F = [0, 1]
F.append(new)

# 또는 변수 방식
a = 0
b = 1
```

## 회고

* **메모리 효율**: 모든 값을 저장하는 대신 필요한 두 개의 변수만 사용하는 방식을 선택했다.
* **경우 분류의 중요성**: n=0, n=1 케이스를 따로 처리해야 올바른 답을 얻을 수 있었다.
* **변수 업데이트 순서**: `new = a + b`, `a = b`, `b = new` 순서가 중요하다는 것을 배웠다.
* **반복 횟수 계산**: n번째 값을 구하려면 (n-1)번 반복해야 한다는 점을 이해했다.
* **슬라이딩 윈도우 개념**: 두 개의 변수가 마치 창문처럼 한 칸씩 이동하며 계산하는 방식이 인상적이었다.

## 오늘 푼 문제

* **백준**: [1316번 그룹 단어 체커](https://www.acmicpc.net/problem/1316) (Silver 5)

---

## 내 풀이
```python
import sys

N = int(sys.stdin.readline())
count = 0  # 그룹 단어 개수

for _ in range(N):
    word = sys.stdin.readline().strip()
    seen = []  # 이미 등장한 문자들을 저장할 빈 리스트
    is_group_word = True  # 현재 단어가 그룹 단어인지 표시
    
    for i in range(len(word)):  # 단어의 각 문자를 인덱스로 순회
        if i == 0:  # 첫 번째 문자인 경우
            seen.append(word[0])  # 첫 문자는 무조건 seen에 추가
        else:  # 두 번째 문자부터는
            if word[i] == word[i-1]:  # 현재 문자가 직전 문자와 같다면 (연속)
                pass  # 아무것도 하지 않음 (연속이므로 그룹 단어 조건 만족)
            else:  # 현재 문자가 직전 문자와 다르다면
                if word[i] in seen:  # 현재 문자가 이미 seen에 있는지 확인
                    is_group_word = False  # 이미 나왔던 문자 → 그룹 단어 아님
                else:  # 현재 문자가 seen에 없다면 (처음 등장)
                    seen.append(word[i])  # 현재 문자를 seen에 추가
    if is_group_word:  # 단어 검사가 끝난 후, 그룹 단어라면
        count += 1  # 그룹 단어 개수를 1 증가
print(count)  # 최종 그룹 단어의 개수를 출력
```

## 새로 배운 개념

### 1. 그룹 단어의 정의

* 그룹 단어: 같은 문자가 **연속해서만** 나타나는 단어
* 한 문자가 떨어져서 두 번 이상 나오면 그룹 단어가 아니다
```python
# 그룹 단어 예시
"happy"    # X - p가 떨어져서 두 번
"ccazzzzbb" # O - 같은 문자가 연속으로만
"kin"      # O - 모든 문자가 한 번씩만
```

### 2. 입력 처리: sys.stdin.readline()

* `sys.stdin.readline()`은 줄바꿈 문자(`\n`)를 포함해서 입력을 받는다
* `strip()` 메서드로 양끝의 공백과 줄바꿈을 제거한다
```python
word = sys.stdin.readline()    # "happy\n"
word = sys.stdin.readline().strip()  # "happy"
```

### 3. 핵심 알고리즘: 연속성 체크

* **직전 문자와 같은지** 확인하는 것이 핵심이다
* 직전과 같으면 → 연속 → OK
* 직전과 다르면 → 이전에 나온 적 있는지 확인
```python
if word[i] == word[i-1]:
    # 연속! 그룹 단어 조건 만족
else:
    # 연속 아님 → 이전에 나온 적 있는지 확인
    if word[i] in seen:
        # 이미 나왔던 문자 → 그룹 단어 아님!
```

### 4. 인덱스 접근 시 주의사항

* `word[i-1]`을 사용할 때 `i`가 0이면 문제가 발생한다
* 파이썬에서 `-1` 인덱스는 **마지막 요소**를 의미한다
* 따라서 첫 번째 문자(`i == 0`)는 따로 처리해야 한다
```python
# 잘못된 접근
for i in range(len(word)):
    if word[i] == word[i-1]:  # i=0일 때 에러!

# 올바른 접근
for i in range(len(word)):
    if i == 0:
        # 첫 문자 처리
    else:
        if word[i] == word[i-1]:
            # 안전하게 비교
```

### 5. seen 리스트 활용

* `seen` 리스트는 "연속이 끊긴 문자들"을 저장한다
* 첫 문자는 무조건 seen에 추가
* 직전과 다른 문자만 seen에 추가
```python
seen = []  # 빈 리스트로 시작

# "aabbbcb" 처리 과정
seen.append('a')  # ['a']
# a 연속 → 추가 안 함
seen.append('b')  # ['a', 'b']
# b 연속 → 추가 안 함
seen.append('c')  # ['a', 'b', 'c']
# 'b' in seen? → True! 그룹 단어 아님!
```

### 6. 불린 플래그 사용

* `is_group_word` 변수로 그룹 단어 여부를 추적한다
* 처음에는 `True`로 시작
* 문제를 발견하면 `False`로 변경
```python
is_group_word = True  # 일단 그룹 단어라고 가정

# 검사 중...
if word[i] in seen:
    is_group_word = False  # 조건 위반 발견!

# 검사 종료 후
if is_group_word:
    count += 1  # 그룹 단어였다면 카운트
```

### 7. 문제 해결 흐름

**1단계: 입력 받기**
```
N개의 단어 개수 입력
각 단어를 순회하면서 처리
```

**2단계: 각 단어마다 초기화**
```
seen = [] (빈 리스트)
is_group_word = True
```

**3단계: 문자 하나씩 검사**
```
첫 번째 문자(i=0):
  seen에 추가

두 번째 문자부터(i≥1):
  직전 문자와 같은가?
    Yes → pass (연속이므로 OK)
    No → seen에 있는가?
      Yes → is_group_word = False (그룹 단어 아님!)
      No → seen에 추가
```

**4단계: 결과 집계**
```
is_group_word가 True이면:
  count += 1
```

**5단계: 최종 출력**
```
print(count)
```

### 8. 예시로 이해하기

**입력:**
```
3
happy
new
ccazzzzbb
```

**실행 과정:**

**단어 1: "happy"**
```
h (i=0): seen = ['h']
a (i=1): 직전 'h'와 다름 → 'a' in seen? No → seen = ['h', 'a']
p (i=2): 직전 'a'와 다름 → 'p' in seen? No → seen = ['h', 'a', 'p']
p (i=3): 직전 'p'와 같음 → pass
y (i=4): 직전 'p'와 다름 → 'y' in seen? No → seen = ['h', 'a', 'p', 'y']
→ is_group_word = True → count = 1
```

**단어 2: "new"**
```
n (i=0): seen = ['n']
e (i=1): 직전 'n'과 다름 → 'e' in seen? No → seen = ['n', 'e']
w (i=2): 직전 'e'와 다름 → 'w' in seen? No → seen = ['n', 'e', 'w']
→ is_group_word = True → count = 2
```

**단어 3: "ccazzzzbb"**
```
c (i=0): seen = ['c']
c (i=1): 직전 'c'와 같음 → pass
a (i=2): 직전 'c'와 다름 → 'a' in seen? No → seen = ['c', 'a']
z (i=3): 직전 'a'와 다름 → 'z' in seen? No → seen = ['c', 'a', 'z']
z (i=4): 직전 'z'와 같음 → pass
z (i=5): 직전 'z'와 같음 → pass
z (i=6): 직전 'z'와 같음 → pass
b (i=7): 직전 'z'와 다름 → 'b' in seen? No → seen = ['c', 'a', 'z', 'b']
b (i=8): 직전 'b'와 같음 → pass
→ is_group_word = True → count = 3
```

**최종 출력:** `3`

### 9. 주요 실수 포인트

* **인덱스 에러**: `i == 0`일 때 `word[i-1]` 접근하지 않도록 주의
* **순서 실수**: `if word[i] in seen` 순서가 바뀌면 안 됨 (`seen in word[i]` X)
* **불필요한 추가**: 직전과 같은 경우 seen에 추가하면 안 됨
* **조건문 오류**: `else:` 다음에 조건식만 쓰면 안 됨 (`if`를 써야 함)
```python
# 잘못된 코드
else:
    word[i] in seen:  # ← 조건문이 아님!

# 올바른 코드
else:
    if word[i] in seen:
```

## 회고

* **문자열 처리의 기본**을 배웠다. 직전 요소와 비교하는 패턴은 많은 문제에서 활용된다.
* **인덱스 접근 시 경계 조건**을 항상 고려해야 한다는 것을 배웠다. 특히 `i-1` 같은 접근은 첫 요소에서 조심해야 한다.
* **리스트를 활용한 집합 연산**(`in` 연산자)을 자연스럽게 사용할 수 있게 되었다.
* **불린 플래그 패턴**은 조건을 추적하는 깔끔한 방법이다. 많은 알고리즘 문제에서 활용할 수 있을 것 같다.

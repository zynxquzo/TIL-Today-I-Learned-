# 오늘의 알고리즘 (2026.02.28)

## 오늘 푼 문제

* **백준**: [18352번 특정 거리의 도시 찾기](https://www.acmicpc.net/problem/18352) (Silver 2)

---

## 내 풀이
```python
import sys
input = sys.stdin.readline
from collections import deque

N, M, K, X = map(int, input().split())
graph = []
for _ in range(N+1):
    graph.append([])

for _ in range(M):
    A, B = map(int, input().split())
    graph[A].append(B)
    
distance = [-1] * (N+1)
distance[X] = 0

queue = deque([X])

while queue:
    now = queue.popleft()
    for next_city in graph[now]:
        if distance[next_city] == -1:
            distance[next_city] = distance[now] + 1
            queue.append(next_city)
            
result = []
for i in range(1, N+1):
    if distance[i] == K:
        result.append(i)

if result:
    result.sort()
    for city in result:
        print(city)
else:
    print(-1)
```

## 새로 배운 개념

### 1. 입력 처리의 기본

* `map(int, input().split())`을 사용하여 여러 정수를 한 번에 입력받는다.
* `input().split()`은 공백으로 구분된 문자열을 리스트로 만든다.
* `map(int, ...)`은 각 문자열을 정수로 변환한다.
```python
# "5 6 2 1" 입력 시
N, M, K, X = map(int, input().split())
# N=5, M=6, K=2, X=1
```

### 2. 그래프 표현 방법 (인접 리스트)

* 그래프는 **인접 리스트** 방식으로 표현한다.
* 각 도시에서 갈 수 있는 도시들을 리스트로 저장한다.
* 0번 인덱스는 사용하지 않고 1번부터 N번까지 사용하여 도시 번호와 인덱스를 일치시킨다.
```python
graph = [[] for _ in range(N+1)]
graph[1] = [2, 3]  # 1번 도시에서 2번, 3번으로 갈 수 있음
graph[2] = [3, 4]  # 2번 도시에서 3번, 4번으로 갈 수 있음
```

### 3. 리스트 곱셈 연산

* `[값] * 개수`를 사용하여 같은 값을 가진 리스트를 쉽게 생성할 수 있다.
```python
distance = [-1] * 5
# 결과: [-1, -1, -1, -1, -1]
```

### 4. BFS로 최단거리 구하기

* BFS는 **출발점으로부터의 최단거리**를 자동으로 보장한다.
* `distance` 리스트에 각 도시까지의 거리를 저장한다.
  * `-1`: 아직 방문하지 않은 도시
  * `0 이상`: 출발 도시로부터의 최단거리
```python
distance = [-1] * (N+1)
distance[X] = 0  # 출발 도시는 거리 0

# BFS 후:
# distance[i] = X번 도시에서 i번 도시까지의 최단거리
```

### 5. BFS의 핵심 로직

* 큐에서 현재 도시를 꺼낸다.
* 현재 도시에서 갈 수 있는 모든 도시를 확인한다.
* 아직 방문하지 않은 도시만 처리한다.
  * 거리 = 현재 도시의 거리 + 1
  * 큐에 추가하여 나중에 방문
```python
while queue:
    now = queue.popleft()
    for next_city in graph[now]:
        if distance[next_city] == -1:  # 미방문
            distance[next_city] = distance[now] + 1
            queue.append(next_city)
```

### 6. 빠른 입출력 처리

* 백준에서 입력이 많은 문제는 `sys.stdin.readline`을 사용해야 시간초과를 피할 수 있다.
* `input()` 함수보다 훨씬 빠르게 입력을 받는다.
```python
import sys
input = sys.stdin.readline  # input을 빠른 버전으로 교체
```

### 7. 문제 해결 흐름

**1단계: 입력 및 그래프 생성**
```
N, M, K, X 입력받기
빈 그래프 생성 (N+1개의 빈 리스트)
M개의 도로 정보를 입력받아 그래프에 추가
```

**2단계: BFS 초기화**
```
distance 리스트를 -1로 초기화
출발 도시 X의 거리를 0으로 설정
큐에 X 추가
```

**3단계: BFS 탐색**
```
큐가 빌 때까지 반복:
  현재 도시를 큐에서 꺼냄
  현재 도시에서 갈 수 있는 도시들 확인
    미방문 도시면:
      거리 = 현재 거리 + 1
      큐에 추가
```

**4단계: 결과 출력**
```
거리가 K인 도시들을 result에 저장
result가 있으면:
  오름차순 정렬 후 한 줄씩 출력
없으면:
  -1 출력
```

### 8. 예시로 이해하기

**입력:**
```
4 4 2 1
1 2
1 3
2 3
2 4
```

**실행 과정:**
```
초기: distance = [-1, 0, -1, -1, -1], queue = [1]

1번 처리: 
  1→2, 1→3 방문
  distance = [-1, 0, 1, 1, -1]
  queue = [2, 3]

2번 처리:
  2→4 방문 (2→3은 이미 방문)
  distance = [-1, 0, 1, 1, 2]
  queue = [3, 4]

3번 처리: 갈 곳 없음
4번 처리: 갈 곳 없음

결과: distance[4] == 2이므로 4 출력
```

## 회고

* **그래프를 인접 리스트로 표현**하는 방법을 알게 되었다. 0번 인덱스를 비워두고 도시 번호와 인덱스를 일치시키는 것이 직관적이었다.
* **BFS의 핵심 원리**를 배웠다. 가까운 곳부터 탐색하기 때문에 자동으로 최단거리가 보장된다.
* BFS의 `while queue` 구조와 `distance[next_city] = distance[now] + 1` 공식이 핵심이었다.

## 오늘 푼 문제

* **백준**: [2775번 부녀회장이 될테야](https://www.acmicpc.net/problem/2775) (Bronze 1)

---

## 내 풀이
```python
# 백준 2775번 - 부녀회장이 될테야

t = int(input())

for _ in range(t):
    k = int(input())  # 찾으려는 층
    n = int(input())  # 찾으려는 호
    
    # 1단계: 2차원 리스트 만들기
    apt = []
    for _ in range(k+1):
        one_floor = [0] * (n+1)
        apt.append(one_floor)
    
    # 2단계: 0층 채우기 (1, 2, 3, 4, ...)
    for i in range(n+1):
        apt[0][i] = i
    
    # 3단계: 1층부터 k층까지 채우기
    for floor in range(1, k+1):
        for room in range(1, n+1):
            # 핵심 규칙: 현재 칸 = 왼쪽 칸 + 아래 칸
            apt[floor][room] = apt[floor][room-1] + apt[floor-1][room]
    
    # 4단계: 답 출력
    print(apt[k][n])
```

## 새로 배운 개념

### 1. 문제의 핵심 규칙 발견

**문제 조건:**
- k층 n호에는 (k-1)층 1호부터 n호까지의 합만큼 사람이 산다

**패턴 분석:**
```
0층: 1, 2, 3, 4, 5
1층: 1, 3, 6, 10, 15
     ↑  ↑  ↑
     1  1+2  1+2+3

2층: 1, 4, 10, 20, 35
     ↑  ↑   ↑
     1  1+3  1+3+6
```

**최적화된 규칙:**
```
k층 n호 = k층 (n-1)호 + (k-1)층 n호
         (왼쪽 칸)    +  (아래 칸)
```

### 2. Bottom-up 방식의 DP

* **Bottom-up**: 작은 문제부터 차례로 해결해 나가는 방식
* **장점**: 재귀 깊이 제한 걱정 없음, 반복문으로 구현
```python
# 0층부터 시작 (이미 알고 있는 값)
for i in range(n+1):
    apt[0][i] = i

# 1층, 2층, ... k층 순서로 계산
for floor in range(1, k+1):
    for room in range(1, n+1):
        apt[floor][room] = apt[floor][room-1] + apt[floor-1][room]
```

### 3. DP 테이블 채우기 과정

**입력: k=2, n=3**

**초기 상태:**
```
apt = [
    [0, 0, 0, 0],  # 0층
    [0, 0, 0, 0],  # 1층
    [0, 0, 0, 0]   # 2층
]
```

**0층 채우기:**
```
apt = [
    [0, 1, 2, 3],  # 0층 완성
    [0, 0, 0, 0],  # 1층
    [0, 0, 0, 0]   # 2층
]
```

**1층 채우기:**
```
1층 1호: apt[1][0] + apt[0][1] = 0 + 1 = 1
1층 2호: apt[1][1] + apt[0][2] = 1 + 2 = 3
1층 3호: apt[1][2] + apt[0][3] = 3 + 3 = 6

apt = [
    [0, 1, 2, 3],  # 0층
    [0, 1, 3, 6],  # 1층 완성
    [0, 0, 0, 0]   # 2층
]
```

**2층 채우기:**
```
2층 1호: apt[2][0] + apt[1][1] = 0 + 1 = 1
2층 2호: apt[2][1] + apt[1][2] = 1 + 3 = 4
2층 3호: apt[2][2] + apt[1][3] = 4 + 6 = 10

apt = [
    [0, 1, 2, 3],   # 0층
    [0, 1, 3, 6],   # 1층
    [0, 1, 4, 10]   # 2층 완성

```

**답 출력:**
```python
print(apt[2][3])  # 10
```

### 4. 반복문 구조 이해

**이중 반복문의 역할:**
```python
for floor in range(1, k+1):      # 바깥 반복문: 층 이동
    for room in range(1, n+1):   # 안쪽 반복문: 호수 이동
        # 계산
```

**실행 순서:**
```
floor=1, room=1 → 1층 1호 계산
floor=1, room=2 → 1층 2호 계산
floor=1, room=3 → 1층 3호 계산
floor=2, room=1 → 2층 1호 계산
floor=2, room=2 → 2층 2호 계산
floor=2, room=3 → 2층 3호 계산
```

### 5. 연산자 우선순위

* **Python 연산 순서**: `*` (곱하기) > `+` (더하기)
* **잘못된 코드**:
```python
[0] * n+1  # ([0] * n) + 1 → 에러!
```

* **올바른 코드**:
```python
[0] * (n+1)  # n+1을 먼저 계산 → [0, 0, 0, ...]
```

### 6. 왜 (k+1)과 (n+1)을 사용할까?

**층 개수:**
- 0층, 1층, 2층, 3층 → 4개 층
- 0부터 k까지 → (k+1)개

**호수 개수:**
- 0호(안 씀), 1호, 2호, 3호 → 4칸 필요
- 0부터 n까지 → (n+1)개
```python
range(k+1)  # 0, 1, 2, ..., k
range(n+1)  # 0, 1, 2, ..., n
```

### 7. DP 문제 해결 패턴

**1단계: 규칙 찾기**
```
작은 예시로 손으로 계산
→ 패턴 발견
```

**2단계: 점화식 세우기**
```
현재 값 = 이전 값들의 조합
→ apt[i][j] = apt[i][j-1] + apt[i-1][j]
```

**3단계: 초기값 설정**
```
가장 작은 경우의 값 미리 저장
→ 0층 = [0, 1, 2, 3, ...]
```

**4단계: 반복문으로 채우기**
```
작은 문제부터 큰 문제로
→ 1층 → 2층 → ... → k층
```

### 8. 주요 실수 포인트

* **리스트 크기 실수**: (k+1), (n+1) 대신 k, n 사용
```python
# 잘못된 코드
for _ in range(k):  # 0층~k층은 k+1개!

# 올바른 코드
for _ in range(k+1)
```

* **괄호 누락**: 연산 순서 문제
```python
# 잘못된 코드
[0] * n+1  # 에러 발생

# 올바른 코드
[0] * (n+1)
```

* **인덱스 범위 초과**: 리스트 크기와 접근 범위 불일치
```python
# 잘못된 코드
one_floor = [0] * n      # n개만 생성
for i in range(n+1):     # n+1번 접근 시도 → 에러!

# 올바른 코드
one_floor = [0] * (n+1)  # (n+1)개 생성
for i in range(n+1):     # (n+1)번 접근 → OK!
```

* **점화식 실수**: 잘못된 인덱스 참조
```python
# 잘못된 코드
apt[floor][room] = apt[floor-1][room-1] + apt[floor][room]

# 올바른 코드
apt[floor][room] = apt[floor][room-1] + apt[floor-1][room]
#                  (왼쪽 칸)          + (아래 칸)
```

### 9. DP의 두 가지 방식

**Top-down (메모이제이션):**
- 재귀 함수로 구현
- 필요한 값만 계산
- 이 문제에는 적합하지 않음

**Bottom-up (타뷸레이션) - 이 문제에서 사용:**
- 반복문으로 구현
- 작은 문제부터 순서대로 해결
- 모든 값을 미리 계산
```python
# Bottom-up 방식
for floor in range(1, k+1):
    for room in range(1, n+1):
        apt[floor][room] = apt[floor][room-1] + apt[floor-1][room]
```

### 10. DP 문제 특징

**DP를 사용하는 문제의 신호:**
- "최대/최소 개수"
- "몇 가지 방법"
- "N번째 값 구하기"
- 같은 계산이 반복됨
- 이전 결과를 활용할 수 있음

**DP 문제 풀이 전략:**
1. 작은 예시로 손으로 계산
2. 규칙(점화식) 찾기
3. 초기값 설정
4. 반복문으로 테이블 채우기
5. 필요한 값 출력

## 회고

* **핵심 규칙 발견**: 손으로 계산하며 패턴을 찾아 "현재 칸 = 왼쪽 칸 + 아래 칸"이라는 점화식을 도출했다.
* **Bottom-up 방식**: 0층부터 차례로 채워나가는 방식이 이 문제에 적합했다.
* **인덱스 관리**: 1호부터 시작하는 문제를 0번 인덱스를 포함한 (n+1) 크기 리스트로 해결하면 편리하다.
* **연산자 우선순위 주의**: `[0] * n+1`이 아닌 `[0] * (n+1)`로 괄호를 명확히 해야 한다.

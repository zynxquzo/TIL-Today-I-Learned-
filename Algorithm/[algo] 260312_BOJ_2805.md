## 오늘 푼 문제

* **백준**: [2805번 나무 자르기](https://www.acmicpc.net/problem/2805) (Silver 2)

---

## 내 풀이
```python
import sys

N, M = map(int, sys.stdin.readline().split())  # 나무 수, 필요한 길이
trees = list(map(int, sys.stdin.readline().split()))  # 나무 높이들

def calculate_wood(trees, height):
    total = 0
    for tree in trees:
        if tree > height:
            cut = tree - height
            total += cut
    return total

left = 0
right = max(trees)
answer = 0

while left <= right:
    mid = (left + right) // 2
    wood = calculate_wood(trees, mid)
    
    if wood >= M:
        answer = mid
        left = mid + 1
    else:
        right = mid - 1

print(answer)
```

## 새로 배운 개념

### 1. 이분 탐색 (Binary Search)이란?

* **이분 탐색**: 정렬된 범위에서 값을 빠르게 찾는 알고리즘
* **핵심 아이디어**: 범위를 절반씩 줄여가며 탐색
* **시간 복잡도**: O(log N) - 매우 빠름!
```python
# 선형 탐색 vs 이분 탐색
선형: 1, 2, 3, 4, ..., 1000000 (최악 100만 번)
이분: 500000 → 250000 → ... (최악 20번)
```

### 2. 이분 탐색의 기본 구조
```python
left = 시작값    # 범위의 왼쪽 끝
right = 끝값     # 범위의 오른쪽 끝

while left <= right:  # 범위가 남아있는 동안
    mid = (left + right) // 2  # 중간값 계산
    
    if 조건을 만족:
        # 답 저장 후 범위 조정
    else:
        # 범위 조정
```

### 3. 이분 탐색의 범위 조정

**조건을 만족할 때:**
- 현재 값을 answer에 저장 (일단 답이 될 수 있음)
- 더 나은 답을 찾기 위해 탐색 범위 이동

**조건을 만족하지 않을 때:**
- 현재 값은 답이 될 수 없음
- 반대 방향으로 탐색 범위 이동
```python
if wood >= M:  # 조건 만족
    answer = mid      # 답 후보 저장
    left = mid + 1    # 더 큰 값 탐색
else:              # 조건 불만족
    right = mid - 1   # 더 작은 값 탐색
```

### 4. 매개변수 탐색 (Parametric Search)

* **매개변수 탐색**: 특정 조건을 만족하는 최댓값/최솟값을 찾는 이분 탐색
* 이 문제는 "M 이상의 나무를 얻을 수 있는 **최대 높이**"를 찾는 문제
* 직접 답을 찾는 게 아니라, **가능한 답의 범위를 좁혀가며** 찾음
```python
# 높이 H에 따른 나무 양의 변화
H=0  → 나무 많음 ✅
H=10 → 나무 적당 ✅
H=15 → 나무 딱 맞음 ✅ (답!)
H=16 → 나무 부족 ❌
H=20 → 나무 없음 ❌
```

### 5. 이분 탐색 실행 과정 상세

**입력 예시:**
```
4 7
20 15 10 17
```

**탐색 과정:**
```
[1회차]
left=0, right=20
mid = (0+20)//2 = 10
calculate_wood(trees, 10):
  20-10=10, 15-10=5, 10-10=0, 17-10=7
  total = 22
22 >= 7 ✅
answer = 10, left = 11

[2회차]
left=11, right=20
mid = (11+20)//2 = 15
calculate_wood(trees, 15):
  20-15=5, 15-15=0, 10<15, 17-15=2
  total = 7
7 >= 7 ✅
answer = 15, left = 16

[3회차]
left=16, right=20
mid = (16+20)//2 = 18
calculate_wood(trees, 18):
  20-18=2, 15<18, 10<18, 17<18
  total = 2
2 >= 7 ❌
right = 17

[4회차]
left=16, right=17
mid = (16+17)//2 = 16
calculate_wood(trees, 16):
  20-16=4, 15<16, 10<16, 17-16=1
  total = 5
5 >= 7 ❌
right = 15

[5회차]
left=16, right=15
16 <= 15? ❌ 종료!

최종 답: 15
```

### 6. while left <= right의 의미

* `left <= right`: 탐색할 범위가 남아있는 동안 계속
* `left > right`: 범위가 교차하면 종료
* `=` 포함 이유: left와 right가 같을 때도 확인해야 함
```python
# <= 없으면 마지막 값을 확인 못할 수도 있음
left=17, right=17
while left < right:  # False → 실행 안 됨!
while left <= right: # True → 17 확인 가능!
```

### 7. 나무 양 계산 함수
```python
def calculate_wood(trees, height):
    total = 0
    for tree in trees:
        if tree > height:  # 절단기보다 높은 나무만
            cut = tree - height  # 잘린 부분
            total += cut  # 누적
    return total
```

**동작 원리:**
- 각 나무를 순회하며
- 절단기 높이보다 높은 나무만 잘림
- 잘린 부분(나무높이 - 절단기높이)을 누적

### 8. 이분 탐색 vs 선형 탐색

**선형 탐색 (비효율적):**
```python
for h in range(0, max(trees)+1):
    if calculate_wood(trees, h) >= M:
        answer = h
# 최악: 1,000,000,000번 반복!
```

**이분 탐색 (효율적):**
```python
while left <= right:
    mid = (left + right) // 2
    ...
# 최악: 약 30번 반복!
```

### 9. 문제 해결 흐름

**1단계: 입력 받기**
```
N개의 나무, 필요한 길이 M
나무 높이들을 리스트로 저장
```

**2단계: 이분 탐색 범위 설정**
```
left = 0 (최소 높이)
right = max(trees) (최대 높이)
answer = 0 (초기값)
```

**3단계: 이분 탐색 실행**
```
중간값 계산
해당 높이로 얻을 수 있는 나무 양 계산
조건에 따라 범위 조정
```

**4단계: 결과 출력**
```
저장된 answer 출력
```

### 10. 예시로 이해하기

**입력:**
```
5 20
4 42 40 26 46
```

**그래프로 이해:**
```
나무들:
46 |     *
42 |   *   *
40 |   * * *
26 |   * * * *
 4 |   * * * * *
   +-------------
     1 2 3 4 5

H=36으로 자르면:
46-36=10
42-36=6
40-36=4
26<36=0
4<36=0
total = 20 ✅
```

**탐색 과정:**
```
[0 ~~~~~~ 23 ~~~~~~ 46]  → 23: 64 ✅ → [24 ~ 46]
          [24 ~ 35 ~ 46]  → 35: 27 ✅ → [36 ~ 46]
                [36~41~46]  → 41: 11 ❌ → [36 ~ 40]
                [36~38~40]  → 38: 17 ❌ → [36 ~ 37]
                [36~36~37]  → 36: 20 ✅ → [37 ~ 37]
                      [37]  → 37: 17 ❌ → 종료

답: 36
```

### 11. 주요 실수 포인트

* **while 조건 실수**: `>=` 대신 `<=` 사용해야 함
```python
# 잘못된 코드
while left >= right:  # 범위가 없을 때 실행!

# 올바른 코드
while left <= right:  # 범위가 있을 때 실행!
```

* **answer 타입 실수**: 리스트가 아닌 변수로 저장
```python
# 잘못된 코드
answer = 0
answer.append(mid)  # 에러! 숫자에는 append 불가

# 올바른 코드
answer = 0
answer = mid  # 대입 연산자 사용
```

* **범위 조정 실수**: +1, -1을 빼먹으면 무한 루프
```python
# 잘못된 코드
if wood >= M:
    left = mid  # mid를 다시 확인할 수 있음!

# 올바른 코드
if wood >= M:
    left = mid + 1  # mid+1부터 탐색
```

* **입력 처리 실수**: 나무 높이는 한 줄에 주어짐
```python
# 잘못된 코드
for _ in range(N):
    tree = int(sys.stdin.readline())

# 올바른 코드
trees = list(map(int, sys.stdin.readline().split()))
```

### 12. 이분 탐색 사용 시기

**이분 탐색을 쓸 수 있는 경우:**
- 정렬된 데이터에서 값 찾기
- 최댓값/최솟값 문제 (매개변수 탐색)
- 범위가 매우 큰 경우 (10^9 이상)
- "조건을 만족하는 최대/최소 값" 찾기

**이 문제의 특징:**
- H의 범위: 0 ~ 1,000,000,000 (10억!)
- 선형 탐색 불가능
- H가 클수록 나무가 적어지는 **단조성** 존재
- 매개변수 탐색으로 해결!

## 회고

* **이분 탐색의 핵심**: 범위를 절반씩 줄여가며 효율적으로 탐색한다.
* **매개변수 탐색**: 답을 직접 찾는 게 아니라, 가능한 답의 범위를 좁혀간다.
* **while left <= right**: `=`을 포함해야 모든 경우를 확인할 수 있다.
* **조건 만족 시**: answer에 저장하고, 더 나은 답을 찾기 위해 범위를 이동한다.

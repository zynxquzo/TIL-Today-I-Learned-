# 오늘의 알고리즘 (2026.02.15)

## 오늘 푼 문제

* **프로그래머스 42842번**: [카펫](https://school.programmers.co.kr/learn/courses/30/lessons/42842) (Level 2)

---

## 내 풀이
```python
def solution(brown, yellow):
    # 전체 격자의 수
    total = yellow + brown
    # 가능한 세로 길이를 확인
    for i in range(1, total + 1):
        # i가 약수인지 확인
        if total % i == 0:
            # 세로
            h = i
            # 가로
            w = total // i
            # 노란색 조건 만족하는지 확인
            if (w-2) * (h-2) == yellow:
                answer = [w, h]
                return answer
```

## 새로 배운 개념

### 1. 완전탐색과 약수 활용

* 이 문제는 **완전탐색(Brute Force)** 방식으로 접근한다.
* 전체 격자 수의 약수들을 모두 확인하면서, 조건을 만족하는 가로/세로 조합을 찾는다.
* 약수를 찾는 방법:
  - `total % i == 0`이면 i는 total의 약수
  - 가로 = `total // i`, 세로 = `i`

### 2. 카펫의 구조 이해

**갈색 테두리는 항상 1칸 두께**라는 것이 핵심이다.
```
■ ■ ■ ■ ■ ■  ← 갈색 테두리 (위 1칸)
■ □ □ □ □ ■  ← 갈색 | 노란색 영역 | 갈색
■ □ □ □ □ ■
■ ■ ■ ■ ■ ■  ← 갈색 테두리 (아래 1칸)
```

* 전체 가로가 6칸이면, 노란색 가로는 6 - 2 = 4칸 (양쪽 갈색 각 1칸씩)
* 전체 세로가 4칸이면, 노란색 세로는 4 - 2 = 2칸 (위아래 갈색 각 1칸씩)
* 따라서 **노란색 개수 = (가로 - 2) × (세로 - 2)**

### 3. 문제 해결 과정

**1단계: 전체 격자 수 계산**
```python
total = yellow + brown
```

**2단계: 가능한 가로/세로 조합 찾기**
```python
for i in range(1, total + 1):
    if total % i == 0:
        h = i
        w = total // i
```
* 1부터 전체 격자 수까지 반복하면서 약수를 찾는다
* 약수 i를 세로로, `total // i`를 가로로 설정

**3단계: 조건 확인**
```python
if (w-2) * (h-2) == yellow:
    return [w, h]
```
* 해당 가로/세로에서 계산한 노란색 개수가 실제 yellow와 같은지 확인
* 조건을 만족하면 `[가로, 세로]`를 반환

### 4. 나머지 연산자(%)의 활용

* `a % b == 0`이면 a는 b로 나누어떨어진다 = b는 a의 약수
* 예시:
  - `12 % 3 == 0` → 3은 12의 약수 ✓
  - `12 % 5 == 2` → 5는 12의 약수 아님 ✗

### 5. 정수 나눗셈 연산자(//)

* `//`는 나눗셈의 몫만 반환한다 (소수점 버림)
* 예시:
  - `12 // 3 = 4`
  - `12 // 5 = 2` (12 ÷ 5 = 2.4이지만 정수 부분만)

### 6. 시뮬레이션으로 문제 이해하기
```
brown = 10, yellow = 2
total = 12

i = 1: 12 % 1 == 0 → h = 1, w = 12
  (12-2) × (1-2) = 10 × (-1) = -10 ≠ 2 ✗

i = 2: 12 % 2 == 0 → h = 2, w = 6
  (6-2) × (2-2) = 4 × 0 = 0 ≠ 2 ✗

i = 3: 12 % 3 == 0 → h = 3, w = 4
  (4-2) × (3-2) = 2 × 1 = 2 == 2 ✓
  return [4, 3]
```

## 회고

* **갈색 테두리가 항상 1칸 두께**라는 점을 이해하는 것이 핵심이었다.
* `(w-2) * (h-2) == yellow` 공식이 왜 모든 경우에 통하는지 그림을 그려보니 이해되었다.
* 완전탐색 문제는 모든 경우를 체계적으로 확인하는 것이 중요하다는 것을 깨달았다.

# 오늘의 알고리즘 (2026.02.01)

## 오늘 푼 문제

* **백준 1010번**: [다리 놓기](https://www.acmicpc.net/problem/1010) (Silver 5)

---

## 내 풀이
```python
import math

T = int(input()) # 테스트 케이스의 개수

for _ in range(T):
    N, M = map(int, input().split())
    result = math.comb(M, N)
    print(result)
```

## 새로 배운 개념

### 1. 조합(Combination) 개념 이해

* N개 중에서 R개를 선택하는 경우의 수를 계산하는 것이 조합이다.
* 공식: nCr = n! / (r! × (n-r)!)
* 다리 놓기 문제는 "M개의 동쪽 사이트 중에서 N개를 선택하는 조합 문제"로 변환할 수 있다는 것을 배웠다.

### 2. Python의 math.comb() 함수 활용

* `math.comb(n, r)` 함수를 사용하면 조합을 간단하게 계산할 수 있다.
* 직접 팩토리얼을 계산하는 것보다 훨씬 간결하고 효율적이다.
* Python 3.8 이상에서 사용 가능하다.

### 3. 문제 패턴 인식의 중요성

* 처음에는 N=1일 때 M가지, N>1일 때 순서대로 다리를 놓는다는 패턴만 발견했다.
* 하지만 이 패턴이 결국 "선택의 문제"라는 것을 깨달으니 조합으로 해결할 수 있었다.

### 4. 반복문 내 출력 위치의 중요성

* 처음에는 `print(result)`를 반복문 밖에 두어서 마지막 결과만 출력되는 실수를 했다.
* 각 테스트 케이스마다 결과를 출력해야 하므로 `print()`는 반복문 안에 있어야 한다.
* 들여쓰기가 로직에 큰 영향을 미친다는 것을 다시 한번 확인했다.

## 회고

* 문제를 보고 바로 "이건 조합이다!"라고 판단하지 못했다. 조합이라는 개념을 까먹고 있었고, 문제와 개념을 연결하는 능력이 부족했다.
* 하지만 생각하는 시간을 가지면서 "M개 중 N개를 선택한다"는 핵심을 파악할 수 있었다.
* 단순히 답을 외우는 것이 아니라, "왜 이 문제가 조합인가?"를 이해하는 것이 중요하다는 것을 배웠다.

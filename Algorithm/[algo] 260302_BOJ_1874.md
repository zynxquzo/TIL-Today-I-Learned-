## 오늘 푼 문제

* **백준**: [1874번 스택 수열](https://www.acmicpc.net/problem/1874) (Silver 2)

---

## 내 풀이
```python
import sys

n = int(sys.stdin.readline())

targets = []
for i in range(n):
    targets.append(int(sys.stdin.readline()))

stack = []
current = 1
result = []

for target in targets:
    if len(stack) == 0 or stack[-1] < target:
        while current <= target:
            stack.append(current)
            result.append('+')
            current += 1
            
    if stack[-1] == target:
        stack.pop()
        result.append('-')
        
    else:
        print('NO')
        break
    
else:
    for op in result:
        print(op)
```

## 새로 배운 개념

### 1. 스택(Stack)의 기본 개념

* 스택은 **LIFO(Last In First Out)** 구조이다.
* 나중에 들어간 것이 먼저 나온다 (접시 쌓기와 같은 원리).
* 파이썬에서는 리스트로 스택을 구현할 수 있다.
```python
stack = []
stack.append(1)  # push: [1]
stack.append(2)  # push: [1, 2]
stack.pop()      # pop: 2가 나오고 [1]만 남음
stack[-1]        # 맨 위 확인: 1
```

### 2. 문제 이해: 목표 수열 만들기

* 1부터 n까지 숫자를 **순서대로만** 스택에 넣을 수 있다.
* 스택에서 숫자를 꺼내서 목표 수열을 만들어야 한다.
* 중요: **1→2→3→4 순서는 바꿀 수 없다!**
```python
# 목표: [4, 3, 6, 8, 7, 5, 2, 1]
# 1,2,3,4 push → 4 pop → 3 pop → 5,6 push → 6 pop ...
```

### 3. 핵심 알고리즘

* **현재 push할 숫자를 추적**하는 `current` 변수가 필요하다.
* 목표 숫자가 스택 맨 위보다 크면 → 목표까지 push
* 목표 숫자가 스택 맨 위와 같으면 → pop
* 목표 숫자가 스택 맨 위보다 작으면 → **불가능!**
```python
current = 1  # 다음에 push할 숫자

for target in targets:
    # 목표까지 push
    while current <= target:
        stack.append(current)
        current += 1
    
    # 스택 맨 위가 목표와 같으면 pop
    if stack[-1] == target:
        stack.pop()
    else:
        # 스택 맨 위 > 목표 → 불가능
        print('NO')
```

### 4. 불가능한 경우 판단

* 스택 맨 위가 목표보다 크면 절대 불가능하다.
* 이유: 이미 큰 숫자가 스택에 있는데, 작은 숫자는 이미 지나갔기 때문!
```python
# 예시: 목표가 3인데 스택 맨 위가 5
# → 3은 이미 push되어 스택 아래에 있거나
# → 아니면 이미 pop되어 사용됨
# → 어느 쪽이든 3을 만들 수 없음!
```

### 5. Python의 for-else 구문

* `for` 문이 `break` 없이 정상 종료되면 `else` 블록이 실행된다.
* 중간에 `break`로 빠져나오면 `else` 블록은 실행되지 않는다.
```python
for i in range(5):
    if i == 3:
        break
else:
    print("정상 종료!")  # 실행 안 됨

for i in range(5):
    pass
else:
    print("정상 종료!")  # 실행됨!
```

### 6. 연산 기록 저장

* `result` 리스트에 `+`와 `-`를 기록한다.
* push할 때마다 `'+'` 추가
* pop할 때마다 `'-'` 추가
* 성공하면 모든 연산을 순서대로 출력
```python
result = []
result.append('+')  # push 기록
result.append('-')  # pop 기록

# 출력
for op in result:
    print(op)
```

### 7. 문제 해결 흐름

**1단계: 입력 받기**
```
n 입력
목표 수열 n개 입력받아 리스트에 저장
```

**2단계: 초기화**
```
빈 스택 생성
current = 1 (다음에 push할 숫자)
빈 result 리스트 생성 (연산 기록)
```

**3단계: 각 목표 숫자 처리**
```
목표 수열의 각 숫자에 대해:
  스택이 비었거나 스택 맨 위 < 목표:
    current부터 목표까지 push
    각 push마다 '+' 기록
  
  스택 맨 위 == 목표:
    pop
    '-' 기록
  
  스택 맨 위 > 목표:
    "NO" 출력하고 종료
```

**4단계: 결과 출력**
```
break 없이 정상 종료:
  result의 모든 연산 출력
```

### 8. 예시로 이해하기

**입력:**
```
8
4 3 6 8 7 5 2 1
```

**실행 과정:**
```
목표: 4
  1,2,3,4 push → stack: [1,2,3,4]
  4 pop → stack: [1,2,3]
  연산: +,+,+,+,-

목표: 3
  stack 맨 위가 3 → 3 pop
  stack: [1,2]
  연산: -

목표: 6
  5,6 push → stack: [1,2,5,6]
  6 pop → stack: [1,2,5]
  연산: +,+,-

목표: 8
  7,8 push → stack: [1,2,5,7,8]
  8 pop → stack: [1,2,5,7]
  연산: +,+,-

목표: 7
  7 pop → stack: [1,2,5]
  연산: -

목표: 5,2,1
  각각 pop → stack: []
  연산: -,-,-

최종 연산: +,+,+,+,-,-,+,+,-,+,+,-,-,-,-,-
```

### 9. 주요 실수 포인트

* `else:` 블록에 불필요한 조건식(`stack[-1] > target:`)을 작성하면 안 된다.
* `else`는 이미 "위의 if가 아닌 경우"를 의미하므로 조건 확인이 필요 없다.
```python
# 잘못된 코드
else:
    stack[-1] > target:  # ← 이건 조건문이 아니라 그냥 식!
    print('NO')

# 올바른 코드
else:
    print('NO')
```

## 회고

* **그리디적 접근**이 핵심이었다. 1부터 순서대로 push하면서, 목표를 만날 때마다 즉시 pop하는 방식으로 해결했다.
* **불가능한 경우를 판단하는 로직**이 중요했다. 스택 맨 위가 목표보다 크면 만들 수 없다는 것을 이해했다.
* **Python의 for-else 구문**을 처음 사용해봤다. break 없이 정상 종료된 경우를 처리하는 깔끔한 방법이었다.

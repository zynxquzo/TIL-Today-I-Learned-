# [TIL] HTML: 구조, 시맨틱 태그, 그리고 Form 활용

## 1. HTML 태그의 기능별 분류

### 1. 구조 및 레이아웃 태그

문서의 전체 골격과 논리적 구역을 정의한다. 단순히 모양을 만드는 것이 아니라, 브라우저와 검색 엔진에 문서의 구조를 설명하는 역할을 한다.

* **기본 구조**: `<html>`, `<head>`, `<body>`를 통해 문서의 시작과 메타데이터, 본문을 구분한다.
* **시맨틱 레이아웃**: 의미를 가지는 태그를 사용하여 유지보수성과 SEO를 높인다.
* `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`, `<footer>`



### 2. 콘텐츠 표시 태그

* **제목(Heading)**: `<h1>`부터 `<h6>`까지 중요도에 따라 사용한다.
* **본문 및 목록**: `<p>`(단락), `<ul>`/`<ol>`(목록), `<li>`(항목)를 사용하여 정보를 구조화한다.
* **멀티미디어**: `<img>`(이미지), `<audio>`, `<video>`를 통해 풍부한 콘텐츠를 삽입한다.

---

## 2. Display 속성에 따른 분류

태그가 화면에 배치되는 방식에 따라 크게 두 가지로 나뉜다.

| 분류 | 특징 | 주요 태그 |
| --- | --- | --- |
| **Block** | 부모 요소의 전체 너비 차지, 줄바꿈 발생, 너비/높이 지정 가능 | `<div>`, `<p>`, `<h1>~<h6>`, `<ul>`, `<li>` |
| **Inline** | 콘텐츠 크기만큼만 너비 차지, 줄바꿈 없음, 너비/높이 지정 불가 | `<span>`, `<a>`, `<strong>`, `<em>` |

---

## 3. 시맨틱(Semantic) 마크업의 중요성

비의미론적 태그(`<div>`, `<span>`)는 스타일링을 위한 컨테이너로 사용하고, 실제 의미가 있는 곳에는 시맨틱 태그를 우선적으로 사용해야 한다.

> **시맨틱 태그의 장점**
> 1. **SEO(검색 엔진 최적화)**: 검색 엔진이 콘텐츠의 중요도를 정확히 파악함.
> 2. **웹 접근성**: 스크린 리더 사용자가 페이지 구조를 쉽게 이해함.
> 3. **가독성**: 개발자가 코드를 볼 때 각 영역의 역할을 한눈에 파악 가능.

## 4. 사용자 입력을 위한 Form 태그

사용자로부터 정보를 입력받아 서버로 전송하는 핵심 요소이다.

### 1. Form의 기본 속성

* `action`: 데이터를 보낼 서버의 URL.
* `method`: 전송 방식 (`GET` - URL에 노출, `POST` - 본문에 숨겨서 전송).

### 2. 주요 입력 요소 (`<input>`)

* **텍스트**: `type="text"`, `type="password"`, `type="email"`.
* **선택**:
* `radio`: 여러 옵션 중 **단 하나**만 선택 (name 속성을 통일해야 함).
* `checkbox`: 여러 옵션을 **중복** 선택 가능.


* **날짜/숫자**: `type="date"`, `type="number"`.

### 3. 기타 양식 태그

* `<textarea>`: 게시글 내용 등 긴 문장을 입력받을 때 사용.
* `<select>` & `<option>`: 드롭다운 형태의 목록 선택.
* `<label>`: `for` 속성을 `input`의 `id`와 연결하여 클릭 영역을 확장하고 접근성을 높임.

## 5. 실습 예제 분석 (Key Examples)

### 1. 예약 폼 구성 (form2.html 참고)

날짜 선택기부터 라디오 버튼까지 복합적인 입력을 처리하는 구조를 학습함.

```html
<div>
    <label for="check_in">체크인:</label>
    <input type="date" name="check_in" id="check_in" />
</div>
<div>
    선호 객실 타입<br>
    <label><input type="radio" name="room" value="standard" /> 스탠다드</label>
    <label><input type="radio" name="room" value="delux" /> 디럭스</label>
</div>

```

### 2. 시맨틱 레이아웃 (basic3.html 참고)

`header`, `nav`, `main`, `aside`, `footer`를 활용하여 현대적인 웹 페이지 구조를 실습함.

## 6. 학습 포인트 (Key Takeaways)

1. **계층적 구조화**: 단순한 텍스트 나열이 아닌, 의미에 맞는 태그를 선택하여 문서의 계층(Hierarchy)을 만드는 것이 중요하다.
2. **사용자 경험(UX) 고려**: `input`에 `placeholder`를 넣거나 `label`을 연결하는 작은 디테일이 사용자의 편의성을 결정한다.
3. **데이터의 무결성**: `required`, `min`, `max` 등의 속성을 활용하여 클라이언트 측에서 1차적인 데이터 검증을 수행할 수 있다.
